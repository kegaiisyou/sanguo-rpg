# 三国江湖 RPG - 版本管理手册

## 一、版本号规则

```
格式：v主版本.功能版本.修正版本
示例：v1.3.0

含义：
- 主版本：重大架构变更（通常不改变）
- 功能版本：新增功能或重大修改
- 修正版本：bug修复或小调整

示例：
- v1.0.0  初始版本
- v1.1.0  添加新功能
- v1.1.1  修复bug
- v2.0.0  重大更新
```

---

## 二、更新流程（必须按顺序执行）

### 每次更新前：检查清单

- [ ] 确认当前本地文件是最新的
- [ ] 确认 GitHub 远程仓库可访问
- [ ] 备份当前版本

### 更新步骤：

**步骤 1：修改代码**
- 直接编辑 `index.html`

**步骤 2：更新版本号**
- 修改 index.html 中的版本号显示
- 更新 VERSION_MANAGEMENT.md 中的版本记录

**步骤 3：运行一键部署**
- 双击 `一键部署.bat`
- 或运行 `一键部署+备份.bat`（推荐）

**步骤 4：确认成功**
- 检查命令行显示 "部署完成！"
- 手机访问测试

---

## 三、备份规范

### 自动备份
运行 `一键部署+备份.bat` 时，自动：
1. 复制 `index.html` → `backup/v1.x.x_index.html`
2. 复制 `VERSION_MANAGEMENT.md` → `backup/v1.x.x_VERSION.md`
3. 上传到 GitHub

### 备份文件命名
```
格式：v{版本号}_{描述}.html
示例：
  v1.3.0_index.html        (游戏主文件)
  v1.3.0_VERSION.md         (版本记录)
  v1.4.0_index.html        (功能更新后备份)
```

## 四、版本记录模板

每次发布新版本，在 `VERSION_MANAGEMENT.md` 中记录：

```markdown
## v1.x.x - YYYY-MM-DD

### 更新内容
- 描述1
- 描述2

### 备份文件
- `v1.x.x_index.html`
```

## 六、紧急回滚

如果新版本有问题：
1. 从 `backup/` 文件夹找到上一个正常版本
2. 恢复 `index.html`
3. 重新运行部署脚本
4. 如果 GitHub 有问题，可以在 GitHub 上找回历史版本

---

## 七、注意事项

1. **不要删除 backup 文件夹** - 包含所有历史版本
2. **每次更新都要备份** - 防止意外丢失
3. **版本号要递增** - 方便追踪
4. **测试后再部署** - 避免带 bug 上线
